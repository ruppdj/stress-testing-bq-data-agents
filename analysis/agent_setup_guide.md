# Programmatic Data Agent Configuration Guide

This guide details how to programmatically configure your BigQuery Conversational Analytics agent (system instructions, verified queries, and glossary terms) using the Google Conversational Analytics REST API. It explains how the automated upload script works and how to maintain or recreate it.

---

## 1. Google Conversational Analytics API Architecture

The BigQuery Studio console uses the Google Conversational Analytics service (API endpoint: `geminidataanalytics.googleapis.com`) under the hood to manage data agents.

### API Resource Path
Data Agents are identified by a hierarchical resource name structure:
`projects/{PROJECT_ID}/locations/{LOCATION}/dataAgents/{AGENT_ID}`

* **`PROJECT_ID`**: The GCP project ID (e.g., `nba-data-agent-testing`).
* **`LOCATION`**: The multi-region or region matching your data source (e.g., `us` for datasets located in the US multi-region).
* **`AGENT_ID`**: The UUID generated by Google when you create the agent (e.g., `agent_0ba1a9f6-c456-48c7-b0cc-eb6052fcc2e0`).

### The Regional Endpoint Gotcha
When managing agents located in a specific region or multi-region (such as `us`), calling the global endpoint (`geminidataanalytics.googleapis.com`) for a write operation returns a **`403 PERMISSION_DENIED`** error:
`"message": "Write access to project 'nba-data-agent-testing' was denied"`

To perform write/update operations on regional agent configurations, you **must** use the regional or multi-regional endpoint subdomains:
* **Multi-regional US endpoint:** `https://geminidataanalytics.us.rep.googleapis.com`
* **Regional endpoints (e.g., us-central1):** `https://geminidataanalytics-us-central1.googleapis.com`

## 2. Programmatic Script Logic (`upload_nba_agent.py`)

The configuration script [upload_nba_agent.py](../scripts/upload_nba_agent.py) automates the upload of the agent's instructions, verified queries, and glossary terms.

### A. Authentication & Tokens
The script uses standard Google Application Default Credentials (ADC) to authenticate:
```python
import google.auth
import google.auth.transport.requests

def get_token():
    # Resolves the local credentials file (~/.config/gcloud/application_default_credentials.json)
    credentials, project = google.auth.default(scopes=['https://www.googleapis.com/auth/cloud-platform'])
    auth_req = google.auth.transport.requests.Request()
    credentials.refresh(auth_req)
    return credentials.token
```
*Note: If the script is run in an automated pipeline (like GitHub Actions), it will automatically resolve the service account credentials from the environment.*

### B. Payload Extraction
1. **System Instructions:** The script parses the markdown file [agent_instructions.md](agent_instructions.md) using a regular expression to extract only the text block enclosed inside the code fence under the `## Instructions (paste into Console)` heading.
2. **Verified Queries Map:** It reads the verified queries list from [verified_queries.json](verified_queries.json). Because the local dbt YAML configuration format differs from Google's API schema, it maps the fields:
   * `question` $\rightarrow$ `naturalLanguageQuestion`
   * `sql` $\rightarrow$ `sqlQuery`
3. **Glossary Terms:** It compiles the list of 8 custom glossary term objects containing the sport-specific definitions (`displayName`, `description`, and `labels` synonyms).

### C. The HTTP PATCH Operation & UpdateMask
To update the agent, the script performs a `PATCH` request to the endpoint:
`PATCH https://geminidataanalytics.us.rep.googleapis.com/v1alpha/projects/{PROJECT_ID}/locations/{LOCATION}/dataAgents/{AGENT_ID}?updateMask={MASK}`

* **Why `updateMask` is critical:** If you make a standard `PATCH` or `PUT` request without an update mask, Google will overwrite the entire agent object, which would wipe out your data source connections (BigQuery tables) configured in the UI. 
* **Selective Updates:** We specify the `updateMask` query parameter to tell Google's API to *only* update the instructions, queries, and glossary fields inside the agent's published context, leaving everything else untouched:
  `updateMask=dataAnalyticsAgent.publishedContext.exampleQueries,dataAnalyticsAgent.publishedContext.systemInstruction,dataAnalyticsAgent.publishedContext.glossaryTerms`

## 3. Required IAM Security Roles

For the script to communicate with the API, the Google account running the terminal (the ADC principal) must have roles specifically authorized to write to Gemini Data Analytics:

* **Roles Required:** Primitive roles (`Owner` or `Editor`) do not automatically inherit newer Gemini/AI permissions. The account must be explicitly granted:
  * **`Gemini Data Analytics Data Agent Owner`** (`roles/geminidataanalytics.dataAgentOwner`) OR
  * **`Gemini Data Analytics Admin`** (`roles/geminidataanalytics.admin`)
* **Role Sync Delay:** When granting these roles in the IAM console, it can take up to 2-3 minutes to propagate across Google's authorization servers.

## 4. Recreating the Setup

If you need to configure a new agent or migrate to a new GCP project:

1. **Obtain the Agent ID:** Create the agent in the BigQuery Studio UI, then open its details page and extract the `agent_...` ID from your browser's address bar.
2. **Configure your Local Files:**
   * Review [agent_instructions.md](agent_instructions.md) to edit instructions.
   * Review [verified_queries.json](verified_queries.json) to update query patterns.
3. **Update the Configuration:** Set `GCP_PROJECT` in your environment and record the agent id in [agent_config.py](../scripts/agent_config.py) (`TARGETS`); `LOCATION`/`API_BASE` live there too.
4. **Execute:** Run the script in the terminal:
   ```bash
   python3 scripts/upload_nba_agent.py
   ```
This programmatically binds all configurations to the target cloud agent.
